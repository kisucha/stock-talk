## 1\. 프로젝트 개요

본 시스템은 박스권 종목의 기술적 분석을 AI와 대화하듯 수행할 수 있는 데스크탑 애플리케이션이다. 안랩(053800.KQ)과 같이 장기 박스권을 유지하면서 이슈 발생 시 급등하는 종목을 발굴하고, OBV/볼린저밴드/RSI 다이버전스 기반의 진입 타이밍 분석을 자연어 대화로 수행한다.

| 항목 | 내용 |
| :---- | :---- |
| 프로젝트명 | 미정 (추후 결정) |
| 개발 환경 | Electron (Windows 데스크탑) |
| AI 엔진 | Ollama (기본) \+ Claude API (중요 판단 시) |
| 데이터베이스 | MariaDB 14 (로컬 네트워크 원격 서버) |
| 데이터 소스 | FinanceDataReader (KRX 전 종목) \+ 수동 입력 |
| 대상 사용자 | 개인 투자자 (본인 및 향후 배포) |

## 2\. 투자 전략 철학

이 시스템이 지원하는 투자 전략의 핵심 원칙:

- **손절 없는 박스권 매매**: 장기 박스권이 확인된 종목에 한해 하단에서 매수 후 상단 or 이슈 발생까지 보유  
- **분할 매수**: 추가 자금을 2\~3회 나눠 진입하여 평균 단가 관리  
- **이슈 대기**: 정치적 이벤트, 실적 개선, 보안 사고 등 촉매 발생 시 박스권 상단 돌파 기대  
- **오후장 집중**: 오전 변동성 소화 후 12시 이후 안정된 흐름에서 진입  
- **일봉 방향성 \+ 단기봉 타이밍**: 일봉으로 큰 그림을 잡고 15분봉/30분봉으로 실제 진입 타이밍 정밀화

- **Gemma3 12B**: 균형 잡힌 성능, 빠른 응답 (VRAM \~10GB)

### 4.2 테이블 구조

**stock\_info (종목 기본 정보)**

CREATE TABLE stock\_info (

  ticker      VARCHAR(20)  PRIMARY KEY,

  name        VARCHAR(100) NOT NULL,

  market      VARCHAR(20),

  box\_low     INT,          \-- 박스권 하단 (수동 설정)

  box\_high    INT,          \-- 박스권 상단 (수동 설정)

  note        TEXT,

  created\_at  TIMESTAMP DEFAULT CURRENT\_TIMESTAMP

);

**stock\_daily (일봉 데이터)**

CREATE TABLE stock\_daily (

  id          INT AUTO\_INCREMENT PRIMARY KEY,

  ticker      VARCHAR(20)  NOT NULL,

  trade\_date  DATE         NOT NULL,

  open        INT          NOT NULL,

  high        INT          NOT NULL,

  low         INT          NOT NULL,

  close       INT          NOT NULL,

  volume      BIGINT       NOT NULL,

  created\_at  TIMESTAMP DEFAULT CURRENT\_TIMESTAMP,

  UNIQUE KEY uq\_ticker\_date (ticker, trade\_date)

);

**chat\_history (AI 대화 기록)**

CREATE TABLE chat\_history (

  id          INT AUTO\_INCREMENT PRIMARY KEY,

  ticker      VARCHAR(20),

  role        ENUM('user','assistant') NOT NULL,

  content     TEXT         NOT NULL,

  engine      ENUM('ollama','claude') DEFAULT 'ollama',

  created\_at  TIMESTAMP DEFAULT CURRENT\_TIMESTAMP

);

## 5\. 기능 설계

### 5.1 기능 목록

| 기능 | 설명 | 우선순위 |
| :---- | :---- | :---- |
| AI 채팅 분석 | 종목 선택 후 AI와 대화하듯 OBV/BB/RSI 분석 | 1순위 |
| 차트 뷰 | 일봉 차트 \+ 지표 시각화 | 1순위 |
| 종목 관리 | 종목 추가/삭제, 박스권 설정 | 1순위 |
| 데이터 수동 입력 | 매일 장 마감 후 오늘 데이터 입력 | 1순위 |
| CSV 일괄 import | 기존 보유 CSV 데이터를 DB로 이전 | 1순위 |
| 박스권 종목 스캔 | KRX 전 종목 중 박스권 패턴 자동 탐색 | 2순위 |
| 데이터 자동 수집 | FinanceDataReader로 매일 자동 업데이트 | 2순위 |
| 매수 알림 | 진입 조건 충족 시 알림 (가격 구역 진입) | 2순위 |
| 백테스트 | 과거 데이터로 전략 수익률 검증 | 3순위 |

### 5.2 AI 분석 채팅 기능 상세

AI가 수행하는 핵심 분석 항목:

- **OBV 추세 분석**: 수급 이탈 여부, MA20 대비 위치  
- **볼린저밴드 분석**: 밴드 폭 확장/수축, %B 위치, 하단/상단 터치 여부  
- **RSI 다이버전스**: 불리시/베어리시 다이버전스, 과매수/과매도 구간  
- **박스권 위치 분석**: 현재가가 설정된 박스권 내 어느 위치인지  
- **진입 시나리오 제안**: 1차/2차 분할 매수가, 진입 조건, 손절 기준  
- **중요 판단 시 Claude API 자동 전환 제안**

### 5.3 박스권 종목 스캔 알고리즘

| 조건 | 기준값 | 의미 |
| :---- | :---- | :---- |
| 고저가 비율 | 52주 고가/저가 \< 2.0배 | 과도한 변동성 제외 |
| 박스권 체류율 | 특정 구간 ±10% 내 체류 60% 이상 | 안정적 박스권 유지 |
| 하단 반등 성공률 | 하단 터치 후 5% 이상 반등 확률 \> 65% | 하단 지지력 검증 |
| 시가총액 | 500억 \~ 1조원 | 유동성 확보 |
| 이슈 급등 히스토리 | 과거 3년 내 2배 이상 상승 이력 | 이슈 발생 시 폭발력 |

## 7\. AI 대화 흐름 예시

### 7.1 일상 분석 (Ollama 기본 모드)

사용자: 안랩 오늘 상태 어때?

AI:  현재가 56,000원, RSI 34.2로 과매도 근접 구간입니다.

     OBV가 MA20 아래에 있어 수급은 아직 약세예요.

     BB %B가 0.04로 하단 근처에 있고, 밴드 폭이 확장 중입니다.

     아직 진입 신호는 나오지 않았어요. 관망이 적절합니다.

사용자: 진입 구역은 어디야?

AI:  1차 진입 구역은 54,000\~55,000원입니다.

     조건: 해당 구간에서 거래량 실린 양봉 1개 확인 후 진입

     2차 진입 구역은 51,000\~52,000원 (2024년 전저점 근처)

     각 구간에서 200만원씩 분할 매수 권장합니다.

### 7.2 중요 판단 시 (Claude API 전환)

사용자: 지금 54,500원인데 들어가도 돼?

AI(Ollama): 가격 구역은 맞습니다. 다만 이 판단은 중요한 매수 결정이므로

            Claude API로 전환해서 더 정밀한 분석을 받아보시겠어요?

            \[Ollama로 계속\] \[Claude API로 전환 (약 20원)\]

→ Claude API 전환 시:

AI(Claude): 현재 54,500원 진입 여부를 종합 분석합니다.

            OBV: 아직 MA20 아래 (수급 미회복)

            RSI: 31.2로 과매도권 진입 직전

            BB: 하단(54,200원) 근처, 지지 테스트 중

            거래량: 직전 봉 대비 1.3배 (아직 충분하지 않음)

            → 진입 조건 70% 충족. 절반 물량(100만원)만 선진입 후

              거래량 확인되면 나머지 100만원 추가 권장합니다.

## 11\. 시스템 프롬프트 설계

시스템 프롬프트는 AI가 매 대화에서 참조하는 핵심 지침이다. 고정된 기본 프롬프트와 상황에 따라 동적으로 주입되는 모드 프롬프트, 그리고 사용자 개인 컨텍스트로 구성된다.

### 11.1 프롬프트 구성 구조

┌─────────────────────────────────────────────┐

│  최종 AI 입력 \= 아래 4개 블록의 조합           │

│                                             │

│  \[A\] 기본 시스템 프롬프트 (고정)              │

│      역할 정의 \+ 분석 철학 \+ 제약 사항         │

│                         ↓                  │

│  \[B\] 상황별 모드 프롬프트 (동적)              │

│      앱이 현재 상황 감지 후 자동 선택           │

│                         ↓                  │

│  \[C\] 개인 컨텍스트 (동적)                    │

│      보유 종목 / 매입가 / 가용자금 등           │

│                         ↓                  │

│  \[D\] 실시간 지표 데이터 (동적)                │

│      DB에서 조회한 최신 지표값                 │

└─────────────────────────────────────────────┘

### 11.2 기본 시스템 프롬프트 \[A\] (고정)

이 프롬프트는 앱 설정에서 수정 가능하며, 모든 대화에 공통으로 적용된다.

\[역할 정의\]

당신은 한국 중소형 박스권 종목 전문 스윙 트레이더 겸 주식 애널리스트입니다.

KOSDAQ/KOSPI 개인투자 10년 이상의 경험을 보유하고 있으며,

다음 전략을 전문으로 합니다:

  \- 장기 박스권 종목 발굴 및 하단 매수 전략

  \- 이슈 드리븐 종목 분석 (정치/보안/테마 이슈)

  \- 손절 없는 분할 매수 전략

  \- 오후장(12:00\~15:30) 진입 타이밍 포착

  \- 안랩(053800.KQ)형 박스권 패턴 인식

\[분석 철학\]

  \- 지표는 보조 도구이며, 전체 맥락과 수급이 우선이다

  \- 확신이 없을 때는 반드시 불확실하다고 명시한다

  \- 불리한 정보도 숨기지 않고 솔직하게 제시한다

  \- 감정적 매매(공포 매도, 탐욕 추격)를 항상 경계하도록 안내한다

  \- 과거 유사 패턴 사례를 근거로 인용한다

  \- 단기 급등락 추격 매수를 지양한다

  \- 전체 투자금의 30% 이상 단일 종목 집중 시 위험을 경고한다

\[답변 언어 및 형식\]

  \- 항상 한국어로 답변한다

  \- 수치는 반드시 포함한다 (예: RSI 34.2, BB %B 0.04)

  \- 판단의 근거를 항상 함께 제시한다

  \- 매수/매도 판단 시 시나리오 A(긍정)와 B(부정)를 모두 제시한다

\[면책 사항\]

  \- 본 분석은 참고용이며, 투자 손실에 대한 책임은 투자자 본인에게 있다

  \- 과거 패턴이 미래를 보장하지 않는다

## 12\. 지표 체계 및 판단 기준

### 12.1 추세 지표

| 지표 | 계산 방법 | 판단 기준 | 활용 |
| :---- | :---- | :---- | :---- |
| 이동평균 (MA) | 5 / 20 / 60 / 120일 단순이동평균 | 정배열(5\>20\>60\>120): 강세 / 역배열: 약세 | 추세 방향 확인 |
| MACD | EMA12 \- EMA26 / 시그널: MACD EMA9 | 골든크로스: 매수 / 데드크로스: 매도 | 추세 전환 포착 |
| 골든/데드크로스 | 5일선과 20일선 교차 | 골든크로스: 단기 매수 / 데드크로스: 단기 매도 | 단기 추세 전환 |

### 12.2 수급 지표

| 지표 | 계산 방법 | 판단 기준 | 활용 |
| :---- | :---- | :---- | :---- |
| OBV | 상승일: \+거래량 / 하락일: \-거래량 누적 | OBV \> MA20: 수급 유입 / OBV \< MA20: 수급 이탈 | 수급 방향 판단 |
| 거래량 비율 | 오늘 거래량 / 20일 평균 | 2배 이상: 이슈 신호 / 0.5배 이하: 관심 저하 | 이슈 강도 판단 |
| VWAP | 당일 (가격×거래량) 합계 / 총 거래량 | 현재가 \> VWAP: 매수세 우위 | 당일 매매 기준 |

### 12.3 변동성 지표

| 지표 | 계산 방법 | 판단 기준 | 활용 |
| :---- | :---- | :---- | :---- |
| 볼린저밴드 | 중심: 20일 MA / 상단: MA+2σ / 하단: MA-2σ | %B \< 0: 하단 이탈 / %B \> 1: 상단 이탈 | 진입/청산 구간 |
| ATR(14) | 최근 14일 평균 진폭 | 손절폭 \= ATR × 1.5\~2배 / 목표가 \= 진입가 \+ ATR × 3배 | 손절/목표가 설정 |
| BB 스퀴즈 | 밴드폭이 최근 20일 최저값 근처 | 스퀴즈 후 확장 방향으로 강한 추세 발생 가능 | 큰 움직임 예측 |

### 12.4 모멘텀 지표

| 지표 | 계산 방법 | 판단 기준 | 활용 |
| :---- | :---- | :---- | :---- |
| RSI(14) | 14일 상승평균 / (상승+하락평균) × 100 | 30 이하: 과매도 / 70 이상: 과매수 | 과매수/과매도, 다이버전스 |
| 스토캐스틱 | K \= (현재가-n일저가)/(n일고가-n일저가)×100 | K \< 20: 과매도 / K \> 80: 과매수 | 단기 모멘텀 |

### 12.5 캔들 패턴

| 패턴 | 형태 | 신호 | 신뢰도 |
| :---- | :---- | :---- | :---- |

| 도지 | 시가 \= 종가 (몸통 없음) | 매수/매도 균형, 추세 전환 가능성 | ★★★ |

| 저점 갭업 | 전일 종가보다 높게 시작 | 강한 매수세 유입 신호 | ★★★★ |

### 12.6 다이버전스 판단 기준

| 종류 | 조건 | 의미 | 신뢰도 |

## 14\. 대화 맥락 관리

### 14.1 맥락 유지 구조

\[맥락 관리 방식\]

1\. DB에 대화 기록 저장 (chat\_history 테이블)

   \- 매 대화 턴마다 role / content / timestamp 저장

2\. 새 질문 시 이전 대화 N턴을 함께 전송

   \- Ollama: 최근 10턴 (토큰 절약)

   \- Claude API: 최근 20턴 (더 긴 맥락 지원)

3\. 맥락 초기화 조건

   \- 종목 변경 시 자동 초기화

   \- 사용자가 '새 대화 시작' 버튼 클릭 시

   \- 24시간 경과 시 자동 초기화

### 14.2 토큰 관리

| 항목 | Ollama | Claude API |
| :---- | :---- | :---- |
| 대화 기록 유지 | 최근 10턴 | 최근 20턴 |
| 시스템 프롬프트 | 기본 \+ 모드 프롬프트 | 기본 \+ 모드 \+ 컨텍스트 전체 |
| 지표 데이터 | 핵심 5개 지표만 | 전체 지표 포함 |
| 예상 토큰 수 | 약 2,000\~3,000 tokens | 약 4,000\~6,000 tokens |
| 예상 비용 | 무료 | 약 15\~30원/회 |
