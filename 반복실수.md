---

| 실수 유형 | 날짜 | 근본 원인 | 방지 절차 |
|-----------|------|-----------|-----------|
| Kiwoom GetCommData 빈값 반환 | 2026-06-15 | pykiwoom이 `rqname`으로 명명한 두 번째 파라미터가 실제론 `sRecordName`임. 대부분 TR의 record name은 `""` (빈 문자열). rqname을 넣으면 항상 빈값. | `OnReceiveTrData` 콜백의 `record` 파라미터를 GetCommData에 직접 전달. 콜백 내부에서 즉시 데이터 추출. |
| GetLoginInfo 타입 오류 | 2026-06-15 | pykiwoom `GetLoginInfo("ACCNO")`가 문자열이 아닌 list 반환. `.split(';')` 호출 시 TypeError. | `isinstance(result, list)` 체크 후 분기 처리. |
| CommRqData 반환값 None | 2026-06-15 | pykiwoom CommRqData에 return 문 없음 → None. `if ret == 0` 조건이 항상 False → QEventLoop 미실행. | ret 체크 없이 바로 `tr_loop.exec_()` 실행. |
| eventsource v4 생성자 오류 | 2026-06-15 | v4 업그레이드 후 exports 구조 변경. `require()` 결과가 클래스가 아닌 객체. | `const { EventSource } = require('eventsource')` 로 destructure. |
| Python subprocess 출력 미표시 | 2026-06-15 | 파이프 출력 시 블록 버퍼링 → 출력 지연/미출력. | `spawn(cmd, ['-u', ...])` — Python `-u` 플래그 필수. |
